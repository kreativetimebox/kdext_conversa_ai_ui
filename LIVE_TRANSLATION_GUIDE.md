# Live Translation — Frontend Integration Guide

How to add **instant, as-you-type translation** (chat, subtitles, voice) to any
frontend using the Voice Gateway. The translation is generated by our own LLM
and streamed back token-by-token over a WebSocket, so the translated text
appears while the user is still typing.

Everything goes through the **gateway** (`:8001`). Never call the LLM service
(`:8008`) directly from a frontend.

---

## 1. What you get

```
user types  ──►  WS /ws/translate (gateway)  ──►  LLM service  ──►  our model
                     ▲                                                  │
   rendered live ◄── │ ◄──────────── streamed tokens ◄──────────────────┘
```

- **Live chat translation** — translate the message box while typing.
- **Live subtitles** — feed STT transcripts (or captions) through the same
  socket and render the stream as subtitle lines.
- **Voice** — pass the finished translation to `POST /api/voice/tts` to speak it.

---

## 2. Connecting

```
wss://<GATEWAY_HOST>/ws/translate?api_key=<USER_API_KEY>
```

| Item | Value |
|---|---|
| URL | `ws://` or `wss://` + gateway host + `/ws/translate` |
| Auth (browser) | `?api_key=` query param — browsers cannot set WS headers |
| Auth (server/native clients) | `X-API-Key` header also accepted |
| Close code `4401` | missing/invalid API key — do **not** auto-reconnect; re-ask for the key |
| Close code `1011` | upstream unreachable — retry with backoff |

The API key is the same per-user `vgw_...` key returned by `POST /signup` /
`POST /login` and used on all other `/api/*` endpoints.

⚠️ The `api_key` appears in the URL — always use `wss://` in production so it
is encrypted in transit, and never log full WS URLs.

---

## 3. Protocol (JSON text frames)

### Client → server

```jsonc
// Ask for a translation. Send one of these on every (debounced) keystroke.
{
  "type": "translate",
  "id": 3,                 // client-chosen, monotonically increasing number
  "text": "hello world",   // max 5000 chars (server truncates)
  "target_lang": "hi",     // ISO code — see §6. Omit for auto ar↔en.
  "engine": "llm"          // "llm" = our model (streams) | "api" = Google (one shot)
}

// Keepalive — send every ~25s so proxies don't drop the idle socket.
{ "type": "ping" }
```

### Server → client

```jsonc
{ "type": "delta", "id": 3, "content": "नम" }   // one token — append to the preview ("llm" engine only)

{ "type": "done", "id": 3,                       // final result (both engines)
  "translation": "नमस्ते दुनिया",
  "source_lang": "en", "target_lang": "hi" }

{ "type": "error", "id": 3, "message": "..." }   // this request failed; socket stays open

{ "type": "pong" }                               // reply to your ping
```

### The two rules that make it feel instant

1. **Newest wins.** The server cancels the in-flight translation whenever a new
   `translate` frame arrives — you don't need to throttle beyond a debounce.
2. **Discard stale frames by `id`.** Frames from an older request can still be
   in flight when you send a new one. Keep a counter, increment it per request,
   and ignore any frame whose `id` isn't the latest.

## 4. Reference implementation (vanilla JS / adapt to React etc.)

```js
const GATEWAY = "https://gateway.example.com";  // your gateway origin
const API_KEY = "vgw_...";                      // per-user key

let ws = null;
let seq = 0;                // request id — only the newest response renders
let debounceTimer = null;
let pingTimer = null;
let reconnectDelay = 1000;  // backoff: 1s → 15s max
let current = { text: "", translation: "", done: false };

function wsUrl() {
  return GATEWAY.replace(/^http/, "ws")
    + "/ws/translate?api_key=" + encodeURIComponent(API_KEY);
}

function connect() {
  if (ws && (ws.readyState === WebSocket.OPEN || ws.readyState === WebSocket.CONNECTING)) return;
  ws = new WebSocket(wsUrl());

  ws.onopen = () => {
    reconnectDelay = 1000;
    clearInterval(pingTimer);
    pingTimer = setInterval(() => {
      if (ws?.readyState === WebSocket.OPEN) ws.send(JSON.stringify({ type: "ping" }));
    }, 25000);
  };

  ws.onmessage = (ev) => {
    const msg = JSON.parse(ev.data);
    if (msg.type === "pong") return;
    if (msg.id !== seq) return;                 // stale — user kept typing

    if (msg.type === "delta") {
      current.translation += msg.content;
      renderPreview(current.translation, /*streaming*/ true);
    } else if (msg.type === "done") {
      current = { ...current, translation: msg.translation, done: true,
                  sourceLang: msg.source_lang, targetLang: msg.target_lang };
      renderPreview(current.translation, false);
    } else if (msg.type === "error") {
      renderError(msg.message);
    }
  };

  ws.onclose = (ev) => {
    clearInterval(pingTimer);
    ws = null;
    if (ev.code === 4401) { promptForApiKey(); return; }  // bad key — don't loop
    setTimeout(connect, reconnectDelay);
    reconnectDelay = Math.min(reconnectDelay * 2, 15000);
  };
}

// Call this from the input's `oninput` handler.
function onType(text, targetLang) {
  clearTimeout(debounceTimer);
  if (!text.trim()) { renderPreview("", false); return; }
  debounceTimer = setTimeout(() => {
    seq += 1;
    current = { text: text.trim(), translation: "", done: false };
    if (ws?.readyState === WebSocket.OPEN) {
      ws.send(JSON.stringify({
        type: "translate", id: seq,
        text: current.text, target_lang: targetLang, engine: "llm",
      }));
    } else {
      connect();
      httpFallback(seq, current.text, targetLang);  // §5 — works even if WS is blocked
    }
  }, 350);  // debounce: translate pauses in typing, not every keystroke
}
```

Rendering tips:
- Put `dir="auto"` on the preview element (RTL languages: Arabic, Urdu, Hebrew…).
- Show a subtle "translating…" state while `delta` frames arrive and a ✓ on `done`.
- On send/commit, reuse `current.translation` if `current.done && current.text`
  matches the input — no second request needed.

## 5. HTTP fallback (WS blocked by proxy/firewall)

Same result, no streaming — the translation arrives in one piece:

```js
async function httpFallback(id, text, targetLang) {
  const resp = await fetch(GATEWAY + "/api/translate", {
    method: "POST",
    headers: { "Content-Type": "application/json", "X-API-Key": API_KEY },
    body: JSON.stringify({ text, target_lang: targetLang, engine: "llm" }),
  });
  const data = await resp.json();       // { translation, source_lang, target_lang }
  if (id !== seq) return;               // stale
  renderPreview(data.translation, false);
}
```

## 6. Language codes (`target_lang`)

Any source language is auto-detected; pick the target from:

`en ar hi bn ta te mr gu kn ml pa or as ur ne si sa kok mai zh es fr pt ru de
ja ko it tr fa id vi th pl nl uk he el sv ro hu cs ms fil sw`

Omitting `target_lang` gives the legacy behaviour: Arabic↔English auto-pair.

`engine`:
- `"llm"` — our model. Streams `delta` frames. Best quality, ~model latency.
- `"api"` — Google's free endpoint. No deltas, single `done` frame. Fastest.

## 7. Speaking the translation (voice)

```js
async function speak(text) {
  const form = new FormData();
  form.append("text", text);            // language + voice are auto-detected
  const resp = await fetch(GATEWAY + "/api/voice/tts", {
    method: "POST", headers: { "X-API-Key": API_KEY }, body: form,
  });
  const audio = new Audio(URL.createObjectURL(await resp.blob()));
  audio.play();                          // streamable via MediaSource for faster start
}
```

For **live subtitles from speech**: record audio → `POST /api/voice/stt`
(multipart `file`) → feed the returned `text` into the same `/ws/translate`
socket → render the stream as a subtitle line.

---

## 8. Checklist

- [ ] Connect with `wss://` + `?api_key=` (query param, not header, in browsers)
- [ ] Debounce input ~300–400 ms; one `translate` frame per pause
- [ ] Increment `id` per request; drop frames whose `id` ≠ latest
- [ ] Append `delta` frames; replace with `translation` on `done`
- [ ] `ping` every 25 s; reconnect with backoff on close (except `4401`)
- [ ] Fall back to `POST /api/translate` when the WS can't connect
- [ ] `dir="auto"` on the output element for RTL languages
- [ ] Reuse the finished live translation on send — don't re-request
